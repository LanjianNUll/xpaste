# 快捷键优先级提升

## 实现方式

已将快捷键注册方式从 Tauri 的 `global-shortcut` 插件改为 Windows 低级键盘钩子（Low-Level Keyboard Hook）。

## 优势

1. **更高优先级**：低级键盘钩子在系统处理按键之前就能拦截，可以覆盖大部分应用级快捷键
2. **阻止传播**：当检测到匹配的快捷键时，会阻止系统继续处理该按键事件
3. **更精确的控制**：可以精确检测修饰键（Ctrl、Alt、Shift、Win）的组合状态

## 限制

即使使用低级键盘钩子，仍然无法覆盖以下快捷键：

- **系统级快捷键**：如 Ctrl+Alt+Del、Win+L 等由操作系统内核处理的快捷键
- **安全相关快捷键**：由 Windows 安全子系统处理的快捷键
- **某些驱动级快捷键**：如显卡驱动、音频驱动注册的快捷键

## 建议

为了避免冲突，建议使用以下快捷键组合：

- `Alt + V`（默认）
- `Alt + C`
- `Ctrl + Shift + V`
- `Win + V`（注意：Windows 10/11 默认用于剪贴板历史，可能需要在系统设置中禁用）

## 技术细节

- 使用 `SetWindowsHookExW` 注册 `WH_KEYBOARD_LL` 类型的钩子
- 在钩子回调中检查按键和修饰键状态
- 匹配成功时返回非零值阻止按键继续传播
- 应用退出时自动清理钩子资源
